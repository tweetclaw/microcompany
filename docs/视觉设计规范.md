# MicroCompany 视觉设计规范

## 文档说明

本文档基于 MicroCompany logo 的设计风格,定义应用的完整视觉设计系统。

**设计理念**: 现代科技感、深色主题、渐变色彩、圆润边角、立体层次

**Logo 特征分析**:
- 三层堆叠的圆角矩形,呈等距视角
- 渐变色彩: 蓝色、粉紫色、橙紫色
- 深蓝色背景 (#0A1628)
- 大圆角设计,现代科技美学

### 4.1 圆角刻度 (Border Radius)


## 12. 实现指南

### 12.1 CSS 变量定义

在 `src/styles/variables.css` 中定义所有设计 token:

```css
:root {
  /* 在这里定义所有上述的 CSS 变量 */
}
```

### 12.2 全局样式

在 `src/styles/global.css` 中应用基础样式:

```css
* {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

body {
  font-family: var(--font-sans);
  font-size: var(--text-base);
  line-height: var(--leading-normal);
  color: var(--text-primary);
  background: var(--bg-primary);
  -webkit-font-smoothing: antialiased;
  -moz-osx-font-smoothing: grayscale;
}
```

### 12.3 组件样式

每个组件使用 CSS Modules 或 styled-components,引用设计 token:

```tsx
// 示例: Button.module.css
.button {
  background: var(--primary-blue);
  padding: var(--padding-sm);
  border-radius: var(--radius-md);
  /* ... */
}
```

## 13. 设计原则

### 13.1 一致性

- 所有组件使用统一的设计 token
- 保持间距、圆角、阴影的一致性
- 统一的交互反馈模式

### 13.2 层次感

- 使用阴影和背景色区分层级
- 重要元素使用渐变和发光效果
- 保持清晰的视觉层次

### 13.3 可读性

- 确保文本与背景有足够对比度
- 使用合适的字体大小和行高
- 避免过度使用渐变影响可读性

### 13.4 性能

- 避免过度使用阴影和模糊效果
- 合理使用动画,避免影响性能
- 优化渐变的使用

---

## 14. 可访问性

### 14.1 颜色对比度

- 主要文本与背景对比度 ≥ 7:1 (AAA 级)
- 次要文本与背景对比度 ≥ 4.5:1 (AA 级)
- 交互元素的焦点状态清晰可见

### 14.2 键盘导航

- 所有交互元素可通过键盘访问
- 焦点状态清晰可见
- 合理的 Tab 顺序

### 14.3 屏幕阅读器

- 使用语义化 HTML
- 提供 ARIA 标签
- 确保动态内容可被朗读
