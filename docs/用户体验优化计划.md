# MicroCompany 用户体验优化计划

**版本**: 1.0  
**创建时间**: 2026-04-15  
**当前阶段**: 阶段 7 - 用户体验优化

---

## 项目背景

MicroCompany 已完成 Claurst 集成（阶段 6），核心功能已验证通过：
- ✅ 工作目录上下文感知（Glob 工具）
- ✅ 文件读取能力（Read 工具）
- ✅ 代码搜索能力（Grep 工具）
- ✅ 命令执行能力（Bash 工具）
- ✅ 文件编辑能力（Edit 工具）
- ✅ 文件创建能力（Write 工具）
- ✅ 流式响应
- ✅ 工具调用可视化

现在进入用户体验优化阶段，目标是提升应用的易用性、稳定性和功能完整性。

---

## 优化目标

### 核心目标
1. **会话持久化** - 保存和恢复对话历史
2. **取消机制** - 支持中断长时间运行的任务
3. **错误处理优化** - 更友好的错误提示和恢复机制
4. **工具调用增强** - 显示更多执行细节

### 次要目标
5. **性能优化** - 改进流式响应性能
6. **UI 细节优化** - 改进交互体验

---

## 实施计划

### 任务 1：会话持久化（优先级：高）

**目标**：保存对话历史到本地，应用重启后可以恢复

**实现方案**：

1. **后端存储层**（`src-tauri/src/storage/mod.rs`）
   - 创建 `ConversationStorage` 结构
   - 使用 JSON 格式存储对话历史
   - 存储路径：`~/.microcompany/conversations/{working_dir_hash}.json`
   - 支持操作：
     - `save_message(working_dir, message)` - 保存单条消息
     - `load_messages(working_dir)` - 加载历史消息
     - `clear_messages(working_dir)` - 清空历史

2. **集成到 ClaurstSession**（`src-tauri/src/claurst/mod.rs`）
   - 在 `new()` 时加载历史消息
   - 在 `send_message()` 后自动保存
   - 添加 `clear_history()` 方法

3. **前端支持**（`src/components/ChatInterface.tsx`）
   - 在 `init_session` 后加载历史消息
   - 添加"清空对话"确认对话框
   - 显示历史消息加载状态

**验收标准**：
- [ ] 发送消息后，重启应用，历史消息正确恢复
- [ ] 切换工作目录，历史消息正确切换
- [ ] 清空对话功能正常工作

**预计时间**：3-4 小时

---

### 任务 2：取消机制（优先级：高）

**目标**：支持中断正在执行的 AI 任务

**实现方案**：

1. **后端取消支持**（`src-tauri/src/claurst/mod.rs`）
   - 使用 `CancellationToken` 支持取消
   - 添加 `cancel_current_task()` 命令
   - 在 `send_message()` 中存储当前的 `cancel_token`

2. **前端取消按钮**（`src/components/InputBox.tsx`）
   - 在加载状态时显示"取消"按钮
   - 点击后调用 `cancel_current_task` 命令
   - 显示取消状态反馈

3. **取消后的状态处理**
   - 标记最后一条消息为"已取消"
   - 清理工具调用状态
   - 恢复输入框可用状态

**验收标准**：
- [ ] 执行长时间任务（如 `npm run build`）时，可以成功取消
- [ ] 取消后，应用状态正确恢复
- [ ] 取消后，可以立即发送新消息

**预计时间**：2-3 小时

---

### 任务 3：错误处理优化（优先级：中）

**目标**：提供更友好的错误提示和恢复机制

**实现方案**：

1. **错误分类和友好提示**（`src-tauri/src/errors.rs`）
   - 创建 `AppError` 枚举，分类错误类型：
     - `ApiError` - API 调用失败（网络、认证等）
     - `ToolError` - 工具执行失败
     - `ConfigError` - 配置错误
     - `SessionError` - 会话错误
   - 为每种错误提供友好的中文提示和建议操作

2. **错误恢复机制**
   - API 错误：显示重试按钮
   - 工具错误：显示错误详情，允许继续对话
   - 配置错误：引导用户检查配置

3. **前端错误显示**（`src/components/ErrorMessage.tsx`）
   - 创建错误消息组件
   - 显示错误类型、原因、建议操作
   - 支持重试、忽略、查看详情

**验收标准**：
- [ ] 网络错误时，显示友好提示和重试按钮
- [ ] 工具执行失败时，显示错误详情
- [ ] 配置错误时，引导用户检查配置

**预计时间**：2-3 小时

---

### 任务 4：工具调用增强（优先级：中）

**目标**：显示更多工具执行细节，提升透明度

**实现方案**：

1. **扩展 ToolCall 类型**（`src/types/index.ts`）
   ```typescript
   export interface ToolCall {
     tool: string;
     action: string;
     status: 'running' | 'success' | 'error';
     result?: string;
     startTime?: number;      // 新增：开始时间
     endTime?: number;        // 新增：结束时间
     duration?: number;       // 新增：执行时长（毫秒）
     details?: string;        // 新增：执行详情
   }
   ```

2. **后端发送详细信息**（`src-tauri/src/claurst/mod.rs`）
   - 在 `tool-call-start` 事件中添加时间戳
   - 在 `tool-call-end` 事件中添加时长和详情
   - 对于 Bash 工具，包含命令输出摘要

3. **前端显示优化**（`src/components/ToolIndicator.tsx`）
   - 显示执行时长
   - 成功时显示简短摘要
   - 失败时显示错误详情
   - 添加展开/收起按钮查看完整输出

**验收标准**：
- [ ] 工具调用显示执行时长
- [ ] Bash 工具显示命令输出摘要
- [ ] 失败时显示详细错误信息

**预计时间**：2-3 小时

---

### 任务 5：性能优化（优先级：低）

**目标**：改进流式响应性能和内存使用

**实现方案**：

1. **流式响应优化**
   - 批量发送消息片段（每 50ms 或 100 字符）
   - 减少事件发送频率

2. **内存优化**
   - 限制历史消息数量（最多保留 100 条）
   - 自动清理旧的工具调用状态

3. **UI 渲染优化**
   - 使用 `React.memo` 优化消息列表渲染
   - 虚拟滚动（如果消息数量很大）

**验收标准**：
- [ ] 流式响应更流畅
- [ ] 长时间使用后，内存占用稳定

**预计时间**：2-3 小时

---

### 任务 6：UI 细节优化（优先级：低）

**目标**：改进交互体验的小细节

**实现方案**：

1. **输入框优化**
   - 支持 Shift+Enter 换行
   - 支持 Ctrl+Enter 发送
   - 自动聚焦输入框

2. **消息列表优化**
   - 自动滚动到最新消息
   - 显示消息时间戳（悬停时）
   - 支持复制消息内容

3. **工作目录选择优化**
   - 记住最近使用的目录
   - 显示目录路径缩写

**验收标准**：
- [ ] 快捷键正常工作
- [ ] 自动滚动流畅
- [ ] 时间戳显示正确

**预计时间**：2-3 小时
