# MicroCompany 网络搜索集成方案

**文档版本**: 1.0  
**创建日期**: 2026-04-15  
**目标**: 为 MicroCompany 添加网络搜索能力，使用 Brave Search API

---

## 一、方案概述

### 1.1 核心目标

为 MicroCompany 添加网络搜索工具，让 AI 能够：
- 搜索实时网络信息
- 获取最新技术文档
- 查找代码示例和解决方案
- 访问当前事件和新闻

### 1.2 技术选型

**搜索引擎**: Brave Search API

**选择理由**:
- ✅ 官方 API，稳定可靠
- ✅ 免费额度充足（2,000 次/月/用户）
- ✅ 无需信用卡即可注册
- ✅ 返回结构化 JSON 数据
- ✅ 支持多种搜索类型（网页、新闻、图片等）
- ✅ 隐私友好，不追踪用户

### 1.3 用户配置模型

**每个用户自行注册和配置**:
- 每个用户注册自己的 Brave Search 账户
- 每个用户获取自己的 API Key
- 每个用户在本地配置文件中配置自己的 Key
- 每个用户独立享有 2,000 次/月的免费额度

**优势**:
- 无需项目方承担 API 费用
- 用户完全控制自己的配额
- 符合 Brave Search 的使用条款
- 扩展性好，不受单一账户限制

## 二、用户指南：注册 Brave Search API

### 2.1 注册步骤

1. **访问 Brave Search API 官网**
   ```
   https://brave.com/search/api/
   ```

2. **点击 "Get Started" 或 "Sign Up"**
   - 使用邮箱注册账户
   - 无需信用卡

3. **创建 API Key**
   - 登录后进入 Dashboard
   - 点击 "Create API Key"
   - 为 Key 命名（例如：MicroCompany）
   - 复制生成的 API Key（格式：`BSA...`）

4. **保存 API Key**
   - ⚠️ 重要：API Key 只显示一次，请妥善保存
   - 建议保存到密码管理器

### 2.2 免费额度说明

- **每月免费请求数**: 2,000 次
- **重置时间**: 每月 1 号
- **超额后**: API 返回 429 错误（配额用尽）
- **升级选项**: 可付费升级到更高额度（可选）

### 2.3 配置到 MicroCompany

1. **打开配置文件**
   ```bash
   # macOS/Linux
   ~/.microcompany/config.json
   
   # Windows
   %USERPROFILE%\.microcompany\config.json
   ```

2. **添加 Brave Search API Key**
   ```json
   {
     "anthropic_api_key": "sk-ant-...",
     "model": "claude-sonnet-4-6",
     "base_url": "https://api.lycloud.top",
     "brave_search_api_key": "BSA..."
   }
   ```

3. **保存并重启 MicroCompany**

### 4.2 WebSearchTool 实现

**文件**: `src-tauri/src/tools/web_search.rs`

```rust
use async_trait::async_trait;
use claurst_tools::{Tool, ToolContext, ToolResult};
use claurst_core::PermissionLevel;
use serde::{Deserialize, Serialize};
use serde_json::{json, Value};
use reqwest;
use anyhow::{Result, anyhow};

/// Brave Search API 响应结构
#[derive(Debug, Deserialize)]
struct BraveSearchResponse {
    web: Option<WebResults>,
}

#[derive(Debug, Deserialize)]
struct WebResults {
    results: Vec<SearchResult>,
}

#[derive(Debug, Deserialize)]
struct SearchResult {
    title: String,
    url: String,
    description: Option<String>,
}

/// WebSearch 工具输入参数
#[derive(Debug, Deserialize)]
struct WebSearchInput {
    query: String,
    #[serde(default = "default_count")]
    count: u32,
}

fn default_count() -> u32 {
    5
}

/// WebSearch 工具实现
pub struct WebSearchTool {
    api_key: String,
}

impl WebSearchTool {
    pub fn new(api_key: String) -> Self {
        Self { api_key }
    }
    
    async fn search(&self, query: &str, count: u32) -> Result<Vec<SearchResult>> {
        let client = reqwest::Client::new();
        
        let response = client
            .get("https://api.search.brave.com/res/v1/web/search")
            .header("Accept", "application/json")
            .header("X-Subscription-Token", &self.api_key)
            .query(&[
                ("q", query),
                ("count", &count.to_string()),
            ])
            .send()
            .await?;
        
        if !response.status().is_success() {
            let status = response.status();
            let body = response.text().await?;
            return Err(anyhow!("Brave Search API error {}: {}", status, body));
        }
        
        let data: BraveSearchResponse = response.json().await?;
        
        Ok(data.web
            .map(|w| w.results)
            .unwrap_or_default())
    }
}

#[async_trait]
impl Tool for WebSearchTool {
    fn name(&self) -> &str {
        "WebSearch"
    }
    
    fn description(&self) -> &str {
        "Search the web for current information, documentation, code examples, and solutions. \
         Use this when you need up-to-date information beyond your knowledge cutoff."
    }
    
    fn permission_level(&self) -> PermissionLevel {
        PermissionLevel::ReadOnly
    }
    
    fn input_schema(&self) -> Value {
        json!({
            "type": "object",
            "properties": {
                "query": {
                    "type": "string",
                    "description": "The search query"
                },
                "count": {
                    "type": "integer",
                    "description": "Number of results to return (default: 5, max: 20)",
                    "default": 5,
                    "minimum": 1,
                    "maximum": 20
                }
            },
            "required": ["query"]
        })
    }
    
    async fn execute(&self, input: Value, _ctx: &ToolContext) -> ToolResult {
        // 解析输入
        let input: WebSearchInput = serde_json::from_value(input)
            .map_err(|e| ToolResult::error(&format!("Invalid input: {}", e)))?;
        
        // 执行搜索
        let results = self.search(&input.query, input.count)
            .await
            .map_err(|e| ToolResult::error(&format!("Search failed: {}", e)))?;
        
        // 格式化结果
        if results.is_empty() {
            return ToolResult::success("No results found.");
        }
        
        let mut output = format!("Found {} results for '{}':\n\n", results.len(), input.query);
        
        for (i, result) in results.iter().enumerate() {
            output.push_str(&format!("{}. {}\n", i + 1, result.title));
            output.push_str(&format!("   URL: {}\n", result.url));
            if let Some(desc) = &result.description {
                output.push_str(&format!("   {}\n", desc));
            }
            output.push('\n');
        }
        
        ToolResult::success(&output)
    }
}
```

## 六、错误处理

### 6.1 常见错误

**1. API Key 未配置**

```
错误: Brave Search API key not configured
解决: 在 ~/.microcompany/config.json 中添加 brave_search_api_key
```

**2. API Key 无效**

```
错误: Brave Search API error 401: Unauthorized
解决: 检查 API Key 是否正确，是否已激活
```

**3. 配额用尽**

```
错误: Brave Search API error 429: Too Many Requests
解决: 等待下月 1 号配额重置，或升级到付费计划
```

**4. 网络错误**

```
错误: Search failed: connection timeout
解决: 检查网络连接，确认可以访问 api.search.brave.com
```

### 6.2 错误处理策略

```rust
async fn execute(&self, input: Value, _ctx: &ToolContext) -> ToolResult {
    match self.search(&input.query, input.count).await {
        Ok(results) => {
            // 格式化结果
            ToolResult::success(&output)
        }
        Err(e) => {
            // 返回友好的错误信息
            let error_msg = if e.to_string().contains("401") {
                "API Key 无效或未激活，请检查配置"
            } else if e.to_string().contains("429") {
                "API 配额已用尽，请等待下月重置或升级计划"
            } else if e.to_string().contains("timeout") {
                "网络连接超时，请检查网络"
            } else {
                "搜索失败，请稍后重试"
            };
            
            ToolResult::error(&format!("{}: {}", error_msg, e))
        }
    }
}
```
