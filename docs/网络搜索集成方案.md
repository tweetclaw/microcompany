# MicroCompany 网络搜索集成方案

**文档版本**: 2.0  
**创建日期**: 2026-04-15  
**更新日期**: 2026-04-16  
**目标**: 为 MicroCompany 添加网络搜索能力，使用 SearXNG

---

## 一、方案概述

### 1.1 核心目标

为 MicroCompany 添加网络搜索工具，让 AI 能够：
- 搜索实时网络信息
- 获取最新技术文档
- 查找代码示例和解决方案
- 访问当前事件和新闻

### 1.2 技术选型

**搜索引擎**: SearXNG (开源元搜索引擎)

**选择理由**:
- ✅ 完全免费，无配额限制
- ✅ 开源项目，社区活跃
- ✅ 聚合多个搜索引擎结果（Google, Bing, DuckDuckGo 等）
- ✅ 返回结构化 JSON 数据
- ✅ 无需注册或 API Key
- ✅ 隐私友好，不追踪用户
- ✅ 可使用公共实例或自托管
- ✅ 开箱即用，用户零配置

**与其他方案对比**:

| 方案 | 费用 | 配额 | 配置复杂度 | 结果质量 |
|------|------|------|-----------|---------|
| SearXNG | 免费 | 无限制 | 零配置 | 良好（聚合多源） |
| Brave Search | 付费 | $5/月≈1000次 | 需注册+API Key | 优秀 |
| Bing Search | 付费 | 1000次/月免费 | 需Azure账户 | 优秀 |
| DuckDuckGo | 免费 | 非官方API | 不稳定 | 良好 |

### 1.3 用户配置模型

**零配置，开箱即用**:
- 应用内置多个稳定的公共 SearXNG 实例
- 用户无需注册任何账户
- 用户无需配置 API Key
- 自动轮询多个实例，确保可用性
- 高级用户可选择自定义实例 URL

**优势**:
- 用户体验最佳，无门槛
- 无需承担任何费用
- 无配额限制
- 支持自托管用户使用自己的实例

## 二、用户指南

### 2.1 基本使用（零配置）

**无需任何配置，直接使用！**

启动 MicroCompany 后，AI 自动具备网络搜索能力。

示例：
```
你: Rust 的 async trait 最新语法是什么？

AI: 让我搜索一下最新的 Rust async trait 语法。
[自动调用 WebSearch]
根据搜索结果，Rust 在 1.75 版本稳定了 async fn in traits 特性...
```

### 2.2 高级配置（可选）

如果你想使用自己的 SearXNG 实例（自托管或特定公共实例），可以在设置中配置。

**方式 1: 通过设置界面**
1. 打开 MicroCompany 设置
2. 进入 "Search Engine" 分区
3. 选择 "Custom SearXNG Instance"
4. 输入实例 URL（例如：`https://searx.be`）
5. 保存设置

**方式 2: 通过配置文件**
编辑 `~/.microcompany/config.json`：
```json
{
  "anthropic_api_key": "sk-ant-...",
  "model": "claude-sonnet-4-6",
  "base_url": "https://api.lycloud.top",
  "searxng_instance": "https://searx.be"
}
```

### 2.3 自托管 SearXNG（可选）

如果你想完全控制搜索服务，可以自己部署 SearXNG。

**使用 Docker 部署**:
```bash
docker run -d \
  --name searxng \
  -p 8080:8080 \
  -v $(pwd)/searxng:/etc/searxng \
  searxng/searxng:latest
```

然后在 MicroCompany 中配置：
```json
{
  "searxng_instance": "http://localhost:8080"
}
```

**详细部署文档**: https://docs.searxng.org/admin/installation.html

### 3.2 数据流

```

    ↓
run_query_loop 检测到 ToolUse { name: "WebSearch" }
    ↓
查找 WebSearchTool
    ↓
执行 tool.execute(input, ctx)
    ├─ 解析搜索参数（query, count）
    ├─ 从配置读取实例 URL（可选）
    ├─ 选择可用的 SearXNG 实例
    ├─ 构建 HTTP 请求
    ├─ 调用 SearXNG API (GET /search?q=...&format=json)
    ├─ 解析 JSON 响应
    └─ 格式化为 ToolResult
    ↓
返回搜索结果给 AI
    ↓
AI 基于搜索结果回答用户
```

### 4.2 WebSearchTool 实现

**文件**: `src-tauri/src/tools/web_search.rs`

```rust
use async_trait::async_trait;
use claurst_tools::{Tool, ToolContext, ToolResult};
use claurst_core::PermissionLevel;
use serde::{Deserialize, Serialize};
use serde_json::{json, Value};
use reqwest;
use anyhow::{Result, anyhow};

/// SearXNG API 响应结构
#[derive(Debug, Deserialize)]
struct SearXNGResponse {
    results: Vec<SearchResult>,
}

#[derive(Debug, Deserialize)]
struct SearchResult {
    title: String,
    url: String,
    content: Option<String>,
}

/// WebSearch 工具输入参数
#[derive(Debug, Deserialize)]
struct WebSearchInput {
    query: String,
    #[serde(default = "default_count")]
    count: u32,
}

fn default_count() -> u32 {
    5
}

/// 内置的公共 SearXNG 实例列表
const DEFAULT_INSTANCES: &[&str] = &[
    "https://searx.be",
    "https://searx.work",
    "https://search.bus-hit.me",
    "https://searx.tiekoetter.com",
];

/// WebSearch 工具实现
pub struct WebSearchTool {
    instances: Vec<String>,
    current_instance_index: std::sync::atomic::AtomicUsize,
}

impl WebSearchTool {
    pub fn new(custom_instance: Option<String>) -> Self {
        let instances = if let Some(instance) = custom_instance {
            vec![instance]
        } else {
            DEFAULT_INSTANCES.iter().map(|s| s.to_string()).collect()
        };
        
        Self {
            instances,
            current_instance_index: std::sync::atomic::AtomicUsize::new(0),
        }
    }
    
    fn get_next_instance(&self) -> String {
        let index = self.current_instance_index.fetch_add(1, std::sync::atomic::Ordering::Relaxed);
        self.instances[index % self.instances.len()].clone()
    }
    
    async fn search(&self, query: &str, count: u32) -> Result<Vec<SearchResult>> {
        let client = reqwest::Client::builder()
            .timeout(std::time::Duration::from_secs(10))
            .build()?;
        
        // 尝试多个实例，直到成功
        let mut last_error = None;
        for _ in 0..self.instances.len() {
            let instance = self.get_next_instance();
            
            match self.try_search(&client, &instance, query, count).await {
                Ok(results) => return Ok(results),
                Err(e) => {
                    eprintln!("Failed to search on {}: {}", instance, e);
                    last_error = Some(e);
                }
            }
        }
        
        Err(last_error.unwrap_or_else(|| anyhow!("All SearXNG instances failed")))
    }
    
    async fn try_search(
        &self,
        client: &reqwest::Client,
        instance: &str,
        query: &str,
        count: u32,
    ) -> Result<Vec<SearchResult>> {
        let response = client
            .get(format!("{}/search", instance))
            .query(&[
                ("q", query),
                ("format", "json"),
                ("pageno", "1"),
            ])
            .send()
            .await?;
        
        if !response.status().is_success() {
            let status = response.status();
            return Err(anyhow!("SearXNG API error: {}", status));
        }
        
        let data: SearXNGResponse = response.json().await?;
        
        // 限制返回结果数量
        Ok(data.results.into_iter().take(count as usize).collect())
    }
}

#[async_trait]
impl Tool for WebSearchTool {
    fn name(&self) -> &str {
        "WebSearch"
    }
    
    fn description(&self) -> &str {
        "Search the web for current information, documentation, code examples, and solutions. \
         Use this when you need up-to-date information beyond your knowledge cutoff."
    }
    
    fn permission_level(&self) -> PermissionLevel {
        PermissionLevel::ReadOnly
    }
    
    fn input_schema(&self) -> Value {
        json!({
            "type": "object",
            "properties": {
                "query": {
                    "type": "string",
                    "description": "The search query"
                },
                "count": {
                    "type": "integer",
                    "description": "Number of results to return (default: 5, max: 20)",
                    "default": 5,
                    "minimum": 1,
                    "maximum": 20
                }
            },
            "required": ["query"]
        })
    }
    
    async fn execute(&self, input: Value, _ctx: &ToolContext) -> ToolResult {
        // 解析输入
        let input: WebSearchInput = serde_json::from_value(input)
            .map_err(|e| ToolResult::error(&format!("Invalid input: {}", e)))?;
        
        // 执行搜索
        let results = self.search(&input.query, input.count)
            .await
            .map_err(|e| ToolResult::error(&format!("Search failed: {}", e)))?;
        
        // 格式化结果
        if results.is_empty() {
            return ToolResult::success("No results found.");
        }
        
        let mut output = format!("Found {} results for '{}':\n\n", results.len(), input.query);
        
        for (i, result) in results.iter().enumerate() {
            output.push_str(&format!("{}. {}\n", i + 1, result.title));
            output.push_str(&format!("   URL: {}\n", result.url));
            if let Some(content) = &result.content {
                output.push_str(&format!("   {}\n", content));
            }
            output.push('\n');
        }
        
        ToolResult::success(&output)
    }
}

## 六、错误处理

### 6.1 常见错误

**1. 所有实例不可用**

```
错误: All SearXNG instances failed
解决: 
  - 检查网络连接
  - 尝试配置自定义实例
  - 等待公共实例恢复
```

**2. 自定义实例无法访问**

```
错误: SearXNG API error: 404
解决: 检查配置的实例 URL 是否正确，确认实例正在运行
```

**3. 搜索超时**

```
错误: Search failed: connection timeout
解决: 网络连接不稳定，工具会自动尝试其他实例
```

**4. 无搜索结果**

```
结果: No results found.
说明: 搜索词可能过于具体，尝试使用更通用的关键词
```

### 6.2 错误处理策略

```rust
async fn execute(&self, input: Value, _ctx: &ToolContext) -> ToolResult {
    match self.search(&input.query, input.count).await {
        Ok(results) => {

            } else if e.to_string().contains("404") {
                "配置的 SearXNG 实例不存在或不可访问"
            } else {
                "搜索失败，请稍后重试"
            };
            
            ToolResult::error(&format!("{}: {}", error_msg, e))
        }
    }
}

## 六、错误处理

### 6.1 常见错误

**1. API Key 未配置**

```
错误: Brave Search API key not configured
解决: 在 ~/.microcompany/config.json 中添加 brave_search_api_key
```

**2. API Key 无效**

```
错误: Brave Search API error 401: Unauthorized
解决: 检查 API Key 是否正确，是否已激活
```

**3. 配额用尽**

```
错误: Brave Search API error 429: Too Many Requests
解决: 等待下月 1 号配额重置，或升级到付费计划
```

**4. 网络错误**

```
错误: Search failed: connection timeout
解决: 检查网络连接，确认可以访问 api.search.brave.com
```

### 6.2 错误处理策略

```rust
async fn execute(&self, input: Value, _ctx: &ToolContext) -> ToolResult {
    match self.search(&input.query, input.count).await {
        Ok(results) => {
            // 格式化结果

            let error_msg = if e.to_string().contains("401") {
                "API Key 无效或未激活，请检查配置"
            } else if e.to_string().contains("429") {
                "API 配额已用尽，请等待下月重置或升级计划"
            } else if e.to_string().contains("timeout") {
                "网络连接超时，请检查网络"
            } else {
                "搜索失败，请稍后重试"
            };
            
            ToolResult::error(&format!("{}: {}", error_msg, e))
        }
    }
}
