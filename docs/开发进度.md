## 项目概述

MicroCompany 是一个基于 Tauri 的 Claude Code GUI 应用，目标是为 Claurst 提供图形界面，同时保留完整的工具调用能力。

---

## 已完成的工作（阶段 1-5）

### ✅ 基础功能（已完成）

1. **完整的 UI 系统**
   - 三段式布局（工具栏、对话区、输入区）
   - Markdown 渲染和代码高亮
   - 基于 logo 的视觉设计系统（深色主题、渐变色彩、圆角设计）

2. **后端命令系统**
   - 会话管理（初始化、状态查询、关闭）
   - 消息处理（基础 API 集成）
   - 配置系统（环境变量 + 配置文件）

3. **Anthropic API 集成**
   - HTTP 客户端（使用 reqwest）
   - 支持 Claude Sonnet 4.6 模型
   - 支持自定义 API 端点
   - 自动配置检测和友好提示

4. **开发工具**
   - [dev.sh](../dev.sh) - 日志管理启动脚本
   - 完整的文档系统

### ⚠️ 当前限制

虽然基础对话功能已经实现，但缺少核心能力：

- ❌ **AI 无法感知工作目录内容** - AI 不知道目录中有什么文件
- ❌ **AI 无法读取或编辑文件** - 无法查看或修改代码
- ❌ **AI 无法执行命令** - 无法运行测试、构建等
- ❌ **没有工具调用能力** - 缺少 Read、Edit、Bash、Glob、Grep 等工具
- ❌ **没有多轮对话上下文** - 每次对话都是独立的

**这意味着**：当前的 MicroCompany 只是一个简单的聊天界面，AI 无法真正帮助你完成编程任务。

## 下一步：Claurst 集成（阶段 6）

### 目标

集成 Claurst 作为核心引擎，使 AI 具备完整的工具调用能力，真正成为一个可用的编程助手。

### 为什么需要 Claurst

Claurst 是 Claude Code 的 Rust 实现，提供了：
- ✅ 完整的工具系统（Read、Edit、Bash、Glob、Grep）
- ✅ 工作目录上下文管理
- ✅ 多轮对话历史
- ✅ 流式响应支持
- ✅ 权限管理

### 集成计划

详见 [开发计划.md](开发计划.md) 的阶段 6，预计需要 **7-11 天**。

主要任务：

## 使用指南

### 启动应用

```bash
# 使用日志管理脚本启动
./dev.sh

# 或使用标准方式
npm run tauri dev
```

### 配置

配置文件位于 `~/.microcompany/config.json`：
```json
{
  "anthropic_api_key": "your-api-key-here",
  "model": "claude-sonnet-4-6",
  "base_url": "https://api.anthropic.com"
}
```

详见：[API配置指南.md](API配置指南.md)

## 最新进展（2026-04-14）

### ✅ 阶段 6.1 完成 - Claurst 架构研究

**已完成的工作**：

1. **深入研究 Claurst 项目结构**
   - 分析了所有 13 个 crates 的职责和依赖关系
   - 理解了核心架构：core → api → tools → query
   - 找到了关键 API 入口点：`run_query_loop()`

2. **理解工具系统**
   - 工具基于 `Tool` trait 实现
   - 统一的权限检查机制
   - 持久化 Shell 状态（cwd + env）
   - 工具结果通过 `ToolResult` 返回

3. **研究会话管理**
   - 会话通过 `QueryConfig` 和 `ToolContext` 初始化
   - 工作目录通过 `ToolContext.working_dir` 传递
   - 对话历史存储在 `Vec<Message>` 中

4. **分析配置系统**
   - Settings 从 `~/.claurst/settings.json` 加载
   - 支持环境变量和自定义 API 端点
   - 权限模式：Auto、Interactive、Restricted

5. **创建文档**
   - ✅ [Claurst架构分析.md](Claurst架构分析.md) - 完整的架构文档
   - ✅ [Claurst集成方案.md](Claurst集成方案.md) - 详细的集成方案

6. **验证编译**
   - ✅ Claurst workspace 编译成功
   - ✅ 所有 crates 无错误（仅有少量警告）

**关键发现**：

- Claurst 使用模块化的 crate 架构，易于集成
- 工具系统设计优雅，基于 trait 的插件架构
- 流式响应通过 `AnthropicStreamEvent` 处理
- 权限管理灵活，支持自动模式
- 配置系统兼容我们现有的配置
