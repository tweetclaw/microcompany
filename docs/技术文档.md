# MicroCompany 技术文档

## 1. 技术目标

为 Claurst 创建一个图形界面版本，让用户通过 GUI 而不是命令行与 Claude 对话。

## 2. 技术方案

### 2.1 架构（最佳方案）

```
┌─────────────────────────────────────────┐
│   MicroCompany (Tauri + React)          │
│                                         │
│  ┌──────────────────────────────┐      │
│  │   GUI 对话界面 (React)       │      │
│  └──────────┬───────────────────┘      │
│             │                           │
│             │ Tauri Commands            │
│             │                           │
│  ┌──────────▼───────────────────┐      │
│  │   Rust Backend (Tauri)       │      │
│  │                              │      │
│  │   直接引用 Claurst crates:   │      │
│  │   - claurst-core             │      │
│  │   - claurst-query            │      │
│  │   - claurst-commands         │      │
│  └──────────┬───────────────────┘      │
└─────────────┼───────────────────────────┘
              │
              │ 通过 Claurst 调用
              │
┌─────────────▼───────────────────────────┐
│   Claude API                            │
└─────────────────────────────────────────┘
```

### 2.2 核心原则

1. **直接集成 Claurst crates** - 在 Tauri 的 Rust 侧引用 Claurst 的 Rust 库
2. **完全复用能力** - 所有 AI 能力、工具调用、配置管理都来自 Claurst
3. **只做 GUI 层** - 我们只负责图形界面，不重新实现任何核心逻辑
4. **保持简单** - MVP 只做对话界面，其他功能后续添加

### 2.3 为什么这是最佳方案

**优势：**
- ✅ 完全复用 Claurst 的所有能力（多模型、工具调用、配置等）
- ✅ 深度集成，可以获取细粒度的事件和状态
- ✅ 技术栈一致（Tauri 和 Claurst 都是 Rust）
- ✅ 维护成本低（Claurst 更新时只需更新依赖）
- ✅ 不需要进程通信或输出解析

**对比其他方案：**
- 比 stdin/stdout 方案：更稳定、更强大、更易维护
- 比直接调用 Claude API：不需要重新实现工具调用等复杂逻辑

### 4.3 Rust 侧：封装 Claurst

```rust
// src-tauri/src/claurst_wrapper.rs
use claurst_core::{Agent, Config};
use serde::{Deserialize, Serialize};

#[derive(Serialize, Deserialize)]
pub struct Message {
    pub role: String,
    pub content: String,
}

pub struct ClaurstSession {
    agent: Agent,
    history: Vec<Message>,
}

impl ClaurstSession {
    pub fn new(working_dir: &str) -> Result<Self, Box<dyn std::error::Error>> {
        // 读取 Claurst 配置
        let config = Config::load_from_default_path()?;
        
        // 创建 Agent 实例
        let agent = Agent::new(config, working_dir)?;
        
        Ok(Self {
            agent,
            history: Vec::new(),
        })
    }
    
    pub async fn send_message(&mut self, content: &str) -> Result<String, Box<dyn std::error::Error>> {
        // 添加用户消息到历史
        self.history.push(Message {
            role: "user".to_string(),
            content: content.to_string(),
        });
        
        // 调用 Claurst 处理消息
        let response = self.agent.process_message(content).await?;
        
        // 添加助手回复到历史
        self.history.push(Message {
            role: "assistant".to_string(),
            content: response.clone(),
        });
        
        Ok(response)
    }
    
    pub fn get_history(&self) -> &[Message] {
        &self.history
    }
}
```

### 4.4 Rust 侧：Tauri 命令

```rust
// src-tauri/src/commands.rs
use tauri::State;
use tokio::sync::Mutex;
use crate::claurst_wrapper::{ClaurstSession, Message};

pub struct AppState {
    pub session: Mutex<Option<ClaurstSession>>,
}

#[tauri::command]
pub async fn init_session(
    working_dir: String,
    state: State<'_, AppState>,
) -> Result<(), String> {
    let session = ClaurstSession::new(&working_dir)
        .map_err(|e| e.to_string())?;
    
    *state.session.lock().await = Some(session);
    Ok(())
}

#[tauri::command]
pub async fn send_message(
    message: String,
    state: State<'_, AppState>,
) -> Result<String, String> {
    let mut session_guard = state.session.lock().await;
    let session = session_guard.as_mut()
        .ok_or("Session not initialized")?;
    
    session.send_message(&message)
        .await
        .map_err(|e| e.to_string())
}

#[tauri::command]
pub async fn get_history(
    state: State<'_, AppState>,
) -> Result<Vec<Message>, String> {
    let session_guard = state.session.lock().await;
    let session = session_guard.as_ref()
        .ok_or("Session not initialized")?;
    
    Ok(session.get_history().to_vec())
}
```

### 4.5 前端：对话界面

```typescript
// src/App.tsx
import { useState, useEffect } from 'react';
import { invoke } from '@tauri-apps/api/core';

interface Message {
  role: string;
  content: string;
}

function App() {
  const [messages, setMessages] = useState<Message[]>([]);
  const [input, setInput] = useState('');
  const [workingDir, setWorkingDir] = useState('');
  const [isInitialized, setIsInitialized] = useState(false);

  const initSession = async (dir: string) => {
    try {
      await invoke('init_session', { workingDir: dir });
      setIsInitialized(true);
      setWorkingDir(dir);
    } catch (error) {
      console.error('Failed to initialize session:', error);
    }
  };

  const sendMessage = async () => {
    if (!input.trim() || !isInitialized) return;

    try {
      // 添加用户消息到界面
      const userMessage: Message = { role: 'user', content: input };
      setMessages(prev => [...prev, userMessage]);
      setInput('');

      // 发送到 Claurst
      const response = await invoke<string>('send_message', { message: input });

      // 添加助手回复到界面
      const assistantMessage: Message = { role: 'assistant', content: response };
      setMessages(prev => [...prev, assistantMessage]);
    } catch (error) {
      console.error('Failed to send message:', error);
    }
  };

  return (
    <div className="app">
      {!isInitialized ? (
        <div className="init-screen">
          <h1>选择工作目录</h1>
          <input
            type="text"
            value={workingDir}
            onChange={(e) => setWorkingDir(e.target.value)}
            placeholder="/path/to/project"
          />
          <button onClick={() => initSession(workingDir)}>开始</button>
        </div>
      ) : (
        <div className="chat-screen">
          <div className="messages">
            {messages.map((msg, i) => (
              <div key={i} className={`message ${msg.role}`}>
                <div className="role">{msg.role}</div>
                <div className="content">{msg.content}</div>
              </div>
            ))}
          </div>
          <div className="input-area">
            <input
              type="text"
              value={input}
              onChange={(e) => setInput(e.target.value)}
              onKeyPress={(e) => e.key === 'Enter' && sendMessage()}
              placeholder="输入消息..."
            />
            <button onClick={sendMessage}>发送</button>
          </div>
        </div>
      )}
    </div>
  );
}

export default App;
```

## 6. 关键技术挑战

### 6.1 Claurst crates 的 API 理解

**挑战**: 需要理解 Claurst 的内部 API 如何使用

**方案**:
- 阅读 Claurst 源码中的示例
- 查看 Claurst CLI 如何调用这些 crates
- 从最简单的用例开始（单次对话）

### 6.2 异步处理

**挑战**: Claurst 的 API 可能是异步的，需要正确处理

**方案**:
- 使用 Tokio 作为异步运行时
- Tauri 命令使用 async/await
- 前端使用 Promise 处理异步调用

### 6.3 状态管理

**挑战**: 需要在 Tauri 中管理 Claurst 会话状态

**方案**:
- 使用 Tauri 的 State 管理
- 用 Mutex 保护共享状态
- 每个窗口一个会话实例

## 7. 不做的事情

第一版明确不做：

1. **不修改 Claurst 源码** - 只引用，不修改
2. **不重新实现任何 Claurst 功能** - 完全复用
3. **不做复杂的 UI** - 简单的对话界面即可
4. **不做多窗口** - 单窗口单会话
5. **不做配置编辑器** - 使用 Claurst 的配置文件
6. **不做插件系统** - 功能固定

## 8. 验证标准

当以下条件满足时，技术方案验证成功：

1. 能够成功引用和编译 Claurst crates
2. 能够创建 Claurst 会话并发送消息
