# MicroCompany 技术文档

## 1. 技术目标

为 Claurst 创建一个图形界面版本，让用户通过 GUI 而不是命令行与 Claude 对话。

## 2. 技术方案

### 2.1 架构

```
┌─────────────────────────────────────────┐
│   MicroCompany (Tauri + React)          │
│                                         │
│  ┌──────────────────────────────────┐  │
│  │   GUI 对话界面 (React)           │  │
│  │   - ChatInterface                │  │
│  │   - MessageList                  │  │
│  │   - InputBox                     │  │
│  │   - Toolbar                      │  │
│  └──────────┬───────────────────────┘  │
│             │                           │
│             │ Tauri Commands            │
│             │                           │
│  ┌──────────▼───────────────────────┐  │
│  │   Rust Backend (Tauri)           │  │
│  │                                  │  │
│  │   直接引用 Claurst crates:       │  │
│  │   - claurst-core                 │  │
│  │   - claurst-query                │  │
│  │   - claurst-api                  │  │
│  │   - claurst-tools                │  │
│  └──────────┬───────────────────────┘  │
└─────────────┼───────────────────────────┘
              │
              │ 通过 Claurst 调用
              │
┌─────────────▼───────────────────────────┐
│   Claude API / 其他 AI 提供商           │
└─────────────────────────────────────────┘
```

### 2.2 核心原则

1. **直接集成 Claurst crates** - 在 Tauri 的 Rust 侧引用 Claurst 的 Rust 库
2. **完全复用能力** - 所有 AI 能力、工具调用、配置管理都来自 Claurst
3. **只做 GUI 层** - 我们只负责图形界面，不重新实现任何核心逻辑
4. **保持简单** - MVP 只做对话界面，其他功能后续添加

### 2.3 为什么选择 Claurst

**Claurst 是什么：**
- Kuberwastaken 的 Claurst 是 Claude Code 的 clean-room Rust 重写
- 基于泄露代码的行为规范（spec/）实现，但代码是全新的
- 完整实现了 Claude Code 的所有功能
- 支持 30+ AI 提供商（Anthropic、OpenAI、Google、Ollama 等）
- GPL-3.0 开源，活跃维护

**优势：**
- ✅ 完全复用 Claurst 的所有能力（多模型、工具调用、配置等）
- ✅ 深度集成，可以获取细粒度的事件和状态
- ✅ 技术栈一致（Tauri 和 Claurst 都是 Rust）
- ✅ 维护成本低（Claurst 更新时只需更新依赖）
- ✅ 不需要进程通信或输出解析
- ✅ 合法（clean-room 重写，有法律先例支持）

## 4. 开发方法论

### 4.1 面向 UI 开发（UI First）

**原则：先确定产品的样子，用 UI 约束代码的接口和功能边界**

#### 第一阶段：UI 设计和原型

1. **设计所有界面**
   - 主对话界面
   - 工作目录选择界面
   - 设置界面（后续版本）
   - 错误提示界面

2. **定义所有交互流程**
   - 启动流程
   - 对话流程
   - 错误处理流程
   - 停止生成流程

3. **创建静态原型**
   - 使用 React 创建静态界面
   - 使用假数据填充
   - 验证交互流程
   - 收集用户反馈

#### 第二阶段：从 UI 推导接口

根据 UI 需求，列出所有需要的接口：

```typescript
// 从 UI 推导出的接口需求
interface TauriCommands {
  // 初始化会话（工作目录选择界面需要）
  init_session(workingDir: string): Promise<void>;
  
  // 发送消息（输入框需要）
  send_message(message: string): Promise<string>;
  
  // 获取历史（消息列表需要）
```

**为什么这样做：**
- UI 是用户直接接触的部分，决定了产品的边界
- 先定义 UI 可以避免实现不必要的功能
- UI 驱动接口设计，确保接口符合实际需求
- 接口只实现 UI 需要的功能，不多不少

```rust
#[tauri::command]
pub async fn send_message(
    message: String,
    state: State<'_, AppState>,
) -> Result<String, String> {
    // 真实实现：调用 Claurst
    let mut session_guard = state.session.lock().await;
    let session = session_guard.as_mut()

    session.send_message(&message)
```

### 5.4 Rust 侧：Claurst 封装

```rust
// src-tauri/src/core/claurst_wrapper.rs
use claurst_core::{Config, Session};
use claurst_query::QueryEngine;
use serde::{Deserialize, Serialize};
use anyhow::Result;

#[derive(Serialize, Deserialize, Clone)]
pub struct Message {
    pub role: String,
    pub content: String,
}

pub struct ClaurstSession {
    session: Session,
    query_engine: QueryEngine,
    history: Vec<Message>,
}

impl ClaurstSession {
    pub fn new(working_dir: &str) -> Result<Self> {
        // 读取 Claurst 配置
        let config = Config::load_default()?;
        
        // 创建会话
        let session = Session::new(config, working_dir)?;
        
        // 创建查询引擎
        let query_engine = QueryEngine::new(&session)?;
        
        Ok(Self {
            session,
            query_engine,
            history: Vec::new(),
        })
    }
    
    pub async fn send_message(&mut self, content: &str) -> Result<String> {
        // 添加用户消息
        self.history.push(Message {
            role: "user".to_string(),
            content: content.to_string(),
        });
        
        // 调用 Claurst 处理
        let response = self.query_engine
            .process_query(content)
            .await?;
        
        // 添加助手回复
        self.history.push(Message {
            role: "assistant".to_string(),
            content: response.clone(),
        });
        
        Ok(response)
    }
    
    pub fn get_history(&self) -> &[Message] {
        &self.history
    }
    
    pub fn clear_history(&mut self) {
        self.history.clear();
    }
}
```

### 5.6 前端：自定义 Hooks

```typescript
// src/hooks/useChat.ts
import { useState, useCallback } from 'react';
import { invoke } from '@tauri-apps/api/core';
import { Message } from '../types';

export function useChat() {
  const [messages, setMessages] = useState<Message[]>([]);
  const [isLoading, setIsLoading] = useState(false);
  const [error, setError] = useState<string | null>(null);

  const sendMessage = useCallback(async (content: string) => {
    if (!content.trim()) return;

    setIsLoading(true);
    setError(null);

    // 添加用户消息
    const userMessage: Message = { role: 'user', content };
    setMessages(prev => [...prev, userMessage]);

    try {
      // 调用 Tauri 命令
      const response = await invoke<string>('send_message', { 
        message: content 
      });

      // 添加助手回复
      const assistantMessage: Message = { 
        role: 'assistant', 
        content: response 
      };
      setMessages(prev => [...prev, assistantMessage]);
    } catch (err) {
      setError(err as string);
    } finally {
      setIsLoading(false);
    }
  }, []);

  const clearHistory = useCallback(async () => {
    try {
      await invoke('clear_history');
      setMessages([]);
    } catch (err) {
      setError(err as string);
    }
  }, []);

  return {
    messages,
    isLoading,
    error,
    sendMessage,
    clearHistory,
  };
}
```

### 5.7 前端：主应用

```typescript
// src/App.tsx
import { useState } from 'react';
import { invoke } from '@tauri-apps/api/core';
import { useChat } from './hooks/useChat';
import ChatInterface from './components/ChatInterface';
import DirectoryPicker from './components/DirectoryPicker';

function App() {
  const [workingDir, setWorkingDir] = useState<string>('');
  const [isInitialized, setIsInitialized] = useState(false);
  const chat = useChat();

  const handleInitSession = async (dir: string) => {
    try {
      await invoke('init_session', { workingDir: dir });
      setWorkingDir(dir);
      setIsInitialized(true);
    } catch (error) {
      console.error('Failed to initialize session:', error);
    }
  };

  if (!isInitialized) {
    return (
      <DirectoryPicker onSelect={handleInitSession} />
    );
  }

  return (
    <ChatInterface
      workingDir={workingDir}
      messages={chat.messages}
      isLoading={chat.isLoading}
      error={chat.error}
      onSendMessage={chat.sendMessage}
      onClearHistory={chat.clearHistory}
    />
  );
}

export default App;
```

### 7.2 异步处理

**方案**:
- 使用 Tokio 作为异步运行时
- Tauri 命令使用 async/await
- 前端使用 Promise 处理异步调用
- 正确处理错误和超时

- 用 Mutex 保护共享状态
- 每个窗口一个会话实例
- 正确处理会话生命周期
