# MicroCompany 需求文档

## 1. 一句话需求

把 Claude Code (Claurst) 的命令行对话界面改成图形界面。

## 2. 为什么要做这个

Claude Code 是一个强大的 AI 编程助手，但它是命令行工具。很多用户更习惯图形界面，特别是：

1. 对话历史更容易浏览和查找
2. 代码差异可以更直观地展示
3. 文件操作可以可视化
4. 不需要记忆命令行参数

## 3. 产品定义

MicroCompany 就是 Claude Code 的 GUI 版本。

**它是什么：**
- Claude Code 的图形界面包装
- 保持 Claude Code 的所有能力
- 使用 Claude Code 的配置方式（settings.json）
- 复用 Claude Code 的模型配置

**它不是什么：**
- 不是新的 AI 框架
- 不是多智能体系统
- 不是工作流编排工具
- 不是项目管理工具

## 4. MVP 功能范围

### 4.1 必须做

1. **对话界面**
   - 文本输入框（替代命令行输入）
   - 对话历史显示（替代终端滚动）
   - 支持 Markdown 渲染
   - 代码高亮显示

2. **Claurst 集成**
   - 直接引用 Claurst 的 Rust crates
   - 调用 Claurst 的 API 发送消息
   - 接收并展示 AI 回复
   - 使用 Claurst 现有的配置文件

3. **基础功能**
   - 选择工作目录
   - 清空对话历史
   - 复制代码块
   - 保存对话记录

### 4.2 明确不做

第一版不做,后续版本升级需要实现的功能点：
- 多项目管理
- 角色配置界面
- 工作流画布
- 多 Agent 协作
- 自定义 prompt 编辑器
- 插件系统
- 团队协作功能

## 5. 技术约束

1. **必须使用 Claude Code 作为后端**
   - 不重新实现 AI 对话逻辑
   - 不重新实现工具调用
   - 不重新实现模型配置

2. **必须兼容 Claude Code 的配置**
   - 读取 `~/.claude/settings.json`
   - 使用用户已配置的模型
   - 使用用户已配置的 API keys

3. **必须保持简单**
   - 界面简洁
   - 功能聚焦
   - 易于维护

## 6. 成功标准

当以下条件满足时，MVP 成功：

1. 用户可以通过图形界面与 Claude Code 对话
2. 对话体验不比命令行差
3. 所有 Claude Code 的功能都可以使用
4. 用户不需要学习新的配置方式
5. 至少 3 个真实用户愿意从命令行切换到这个界面

## 7. 产品形态

- 桌面应用（Tauri）
- 单窗口界面
- 左侧：对话历史
- 右侧：输入框和发送按钮
- 顶部：工作目录选择和基础设置

## 8. 开发原则

1. **复用优先** - 能用 Claude Code 的就不自己实现
2. **简单优先** - 功能够用就行，不追求完美
3. **迭代优先** - 先做最小可用版本，再根据反馈迭代
4. **用户优先** - 界面要比命令行更好用，否则没有意义

## 9. 下一步

MVP 验证成功后，可以考虑：
- 多标签页支持多个对话
- 对话历史搜索
- 快捷键支持
- 主题切换

但这些都是 MVP 之后的事情。
