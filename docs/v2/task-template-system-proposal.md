## 1. 需求概述

### 1.1 业务目标

创建可复用的任务模板系统，解决当前每次创建任务都需要手动配置角色和模型的痛点。用户可以预先定义常用的团队结构（如：项目经理、架构师、程序员、测试员、评审员），并为每个角色配置默认的大模型。

### 1.2 核心功能

1. **模板管理**
   - 创建、编辑、删除任务模板
   - 每个模板包含多个预配置的角色
   - 支持系统预置模板和用户自定义模板

2. **角色配置**
   - 为每个角色配置：名称、身份、默认大模型、默认提供商
   - 支持角色排序
   - 角色可以没有配置大模型（创建任务时必须配置）

3. **从模板创建任务**
   - 选择模板快速创建任务
   - 创建前可修改任务名称
   - 创建前可修改每个角色的大模型配置
   - 如果模板中角色未配置模型，创建时强制选择

### 1.3 用户价值

- **效率提升**: 从手动配置 5 个角色减少到选择模板 + 可选调整
- **标准化**: 团队结构标准化，减少配置错误
- **复用性**: 常用团队配置一次定义，多次使用

## 2. 数据模型设计

### 2.1 核心实体

**任务模板 (TaskTemplate)**
- 模板 ID
- 模板名称
- 描述
- 图标
- 是否系统预置
- 创建时间、更新时间

**模板角色 (TemplateRole)**
- 角色 ID
- 所属模板 ID
- 角色名称
- 角色身份（Product Manager, Developer, Tester 等）
- 默认大模型（可为空）
- 默认提供商（可为空）
- 显示顺序
- 创建时间

### 2.2 数据关系

```
TaskTemplate (1) ----< (N) TemplateRole
     |
     | (复制创建)
     v
   Task (1) ----< (N) Role
```

### 2.3 数据库表设计要点

- 模板表需要 `is_system` 字段区分系统预置和用户自定义
- 模板角色表需要 `display_order` 字段支持排序
- 模板角色的 `model` 和 `provider` 字段允许为空
- 使用外键约束保证数据完整性
- 删除模板时级联删除关联的模板角色

## 3. 功能流程设计

### 3.1 创建模板流程

```
用户操作流程：
1. 进入设置 → 任务模板管理
2. 点击"新建模板"
3. 填写模板基本信息（名称、描述、图标）
4. 添加角色：
   - 输入角色名称
   - 选择角色身份（下拉选择）
   - 选择默认大模型（可选）
   - 可添加多个角色
5. 调整角色顺序（拖拽排序）
6. 保存模板

系统处理：
- 验证模板名称不为空
- 验证至少有一个角色
- 生成模板 ID 和角色 ID
- 保存到数据库
- 返回创建成功的模板对象
```

### 3.2 从模板创建任务流程

```
用户操作流程：
1. 点击"新建任务"
2. 选择创建方式：
   - 从空白创建（现有流程）
   - 从模板创建（新功能）
3. 选择模板（显示模板列表和预览）
4. 确认/修改任务名称
5. 查看角色列表：
   - 显示每个角色的默认配置
   - 可修改每个角色的大模型
   - 如果角色未配置模型，必须选择
6. 确认创建

系统处理：
- 读取模板定义
- 验证所有角色都已配置大模型
- 创建任务记录
- 为每个角色创建 Role 和 Session
- 初始化 Claurst 会话
- 返回创建成功的任务对象
```

### 3.3 编辑模板流程

```
用户操作流程：
1. 进入模板管理
2. 选择要编辑的模板
3. 修改模板信息或角色配置
4. 保存更改

系统处理：
- 系统预置模板不允许编辑（只读）
- 用户自定义模板可以编辑
- 更新数据库记录
- 不影响已创建的任务
```

## 6. API 接口设计

### 6.1 模板管理接口

**列出所有模板**
- 功能: 获取系统预置和用户自定义的所有模板
- 返回: 模板列表（包含角色数量统计）

**获取模板详情**
- 功能: 获取指定模板的完整信息
- 参数: 模板 ID
- 返回: 模板对象（包含所有角色配置）

**创建模板**
- 功能: 创建新的用户自定义模板
- 参数: 模板信息 + 角色列表
- 返回: 创建成功的模板对象

**更新模板**
- 功能: 更新用户自定义模板
- 参数: 模板 ID + 更新内容
- 限制: 系统预置模板不可更新
- 返回: 更新后的模板对象

**删除模板**
- 功能: 删除用户自定义模板
- 参数: 模板 ID
- 限制: 系统预置模板不可删除
- 返回: 删除成功确认

### 6.2 任务创建接口

**从模板创建任务**
- 功能: 基于模板创建新任务
- 参数: 
  - 模板 ID
  - 任务名称
  - 角色配置覆盖（可选修改每个角色的模型）
- 验证: 所有角色必须配置大模型
- 返回: 创建成功的任务对象

## 8. 风险评估与缓解

### 8.1 技术风险

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| 模板与现有任务创建流程冲突 | 中 | 低 | 保持向后兼容，手动创建流程仍然可用 |

| 风险 | 影响 | 概率 | 缓解措施 |

| 用户期望模板包含更多配置 | 中 | 中 | 第一版聚焦核心功能，后续迭代扩展 |
