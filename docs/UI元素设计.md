# MicroCompany UI 元素设计

## 文档说明

本文档描述 MicroCompany 的 UI 元素构成、布局和操作逻辑。

**不包含**: 颜色、字体、视觉风格(将在下一阶段设计)

**包含**: 元素定义、布局结构、交互逻辑、状态变化

---

## 1. 整体布局结构

### 1.1 应用窗口

```
┌─────────────────────────────────────────────────────────┐
│  标题栏区域 (TitleBar)                                    │
├─────────────────────────────────────────────────────────┤
│  工具栏区域 (Toolbar)                                     │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  对话区域 (ChatArea)                                     │
│  [占据主要空间,可滚动]                                    │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  输入区域 (InputArea)                                     │
└─────────────────────────────────────────────────────────┘
```

**布局特点:**
- 标题栏: 固定在顶部,高度约 40px
- 工具栏: 固定在标题栏下方,高度约 50px
- 对话区域: 占据剩余空间,可垂直滚动
- 输入区域: 固定在底部,高度自适应(最小 80px,最大 200px)

## 2. 标题栏区域 (TitleBar)

### 2.1 元素构成

```

**元素列表:**

**交互逻辑:**
- 标题栏可拖拽移动窗口

## 3. 工具栏区域 (Toolbar)

### 3.1 元素构成

```
┌─────────────────────────────────────────────────────────┐
│  工作目录: /Users/xxx/project  [选择目录] [清空对话]      │
└─────────────────────────────────────────────────────────┘
```

**元素列表:**

1. **工作目录标签** (WorkingDirLabel)
   - 位置: 左侧
   - 内容: "工作目录:"
   - 功能: 静态标签

2. **工作目录路径** (WorkingDirPath)
   - 位置: 标签右侧
   - 内容: 当前工作目录的完整路径
   - 显示方式: 
     - 如果路径过长,显示省略号
     - 鼠标悬停显示完整路径(tooltip)
   - 功能: 显示当前工作目录

3. **选择目录按钮** (SelectDirButton)
   - 位置: 路径右侧
   - 内容: "选择目录" 或图标
   - 功能: 打开目录选择对话框

4. **清空对话按钮** (ClearChatButton)
   - 位置: 右侧
   - 内容: "清空对话" 或图标
   - 功能: 清空当前对话历史

**交互逻辑:**

- **选择目录按钮点击:**
  1. 打开系统目录选择对话框
  2. 用户选择目录后,更新工作目录路径显示
  3. 调用后端 `init_session` 命令初始化会话
  4. 如果初始化失败,显示错误提示

- **清空对话按钮点击:**
  1. 显示确认对话框: "确定要清空对话历史吗?"
  2. 用户确认后,调用后端 `clear_history` 命令
  3. 清空对话区域的所有消息
  4. 显示提示: "对话已清空"

## 4. 对话区域 (ChatArea)

### 4.1 整体结构

```
┌─────────────────────────────────────────────────────────┐
│  [消息列表,可滚动]                                        │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 用户消息 (UserMessage)                           │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │ AI 消息 (AssistantMessage)                       │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 加载指示器 (LoadingIndicator) [可选]             │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

**布局特点:**
- 消息从上到下排列
- 新消息添加到底部
- 自动滚动到最新消息
- 用户可手动滚动查看历史

### 4.2 用户消息 (UserMessage)

```
┌─────────────────────────────────────────────────────┐
│  👤 User                              [时间戳]       │
│  ┌───────────────────────────────────────────────┐ │
│  │ 消息内容文本                                   │ │
│  │ 支持多行显示                                   │ │
│  └───────────────────────────────────────────────┘ │
│  [复制]                                             │
└─────────────────────────────────────────────────────┘
```

**元素列表:**

1. **用户头像/图标** (UserAvatar)
   - 位置: 左上角
   - 内容: 用户图标或 "👤"
   - 尺寸: 32x32px

2. **角色标签** (RoleLabel)
   - 位置: 头像右侧
   - 内容: "User" 或 "你"
   - 功能: 标识消息发送者

3. **时间戳** (Timestamp)
   - 位置: 右上角
   - 内容: 消息发送时间 (如 "14:30")
   - 格式: HH:mm 或相对时间 (如 "刚刚", "5分钟前")

4. **消息内容** (MessageContent)
   - 位置: 头像下方
   - 内容: 用户输入的文本
   - 显示: 纯文本,保留换行

5. **操作按钮组** (ActionButtons)
   - 位置: 消息内容下方
   - 包含: 复制按钮
   - 显示方式: 默认隐藏,鼠标悬停显示

**交互逻辑:**

- **复制按钮点击:**
  1. 复制消息内容到剪贴板
  2. 按钮文本临时变为 "已复制"
  3. 2秒后恢复为 "复制"

### 4.3 AI 消息 (AssistantMessage)

```
┌─────────────────────────────────────────────────────┐
│  🤖 Assistant                         [时间戳]       │
│  ┌───────────────────────────────────────────────┐ │
│  │ AI 回复内容                                    │ │
│  │ 支持 Markdown 渲染                             │ │
│  │                                               │ │
│  │ ```python                                     │ │
│  │ def hello():                                  │ │
│  │     print("Hello")                            │ │
│  │ ```                                           │ │
│  └───────────────────────────────────────────────┘ │
│  [复制] [重新生成]                                  │
└─────────────────────────────────────────────────────┘
```

**元素列表:**

1. **AI 头像/图标** (AssistantAvatar)
   - 位置: 左上角
   - 内容: AI 图标或 "🤖"
   - 尺寸: 32x32px

2. **角色标签** (RoleLabel)
   - 位置: 头像右侧
   - 内容: "Assistant" 或 "AI"

3. **时间戳** (Timestamp)
   - 位置: 右上角
   - 内容: 消息接收时间

4. **消息内容** (MessageContent)
   - 位置: 头像下方
   - 内容: AI 的回复
   - 渲染: Markdown 格式
     - 支持标题、列表、引用
     - 支持代码块(带语法高亮)
     - 支持粗体、斜体、链接
     - 支持表格

5. **代码块** (CodeBlock) [嵌套在消息内容中]
   - 包含: 语言标签、代码内容、复制按钮
   - 语法高亮: 根据语言类型
   - 复制按钮: 位于代码块右上角

6. **操作按钮组** (ActionButtons)
   - 位置: 消息内容下方
   - 包含: 复制按钮、重新生成按钮
   - 显示方式: 默认隐藏,鼠标悬停显示

**交互逻辑:**

- **复制按钮点击:**
  - 复制整个消息内容(Markdown 原文)

- **代码块复制按钮点击:**
  - 只复制代码内容(不含语言标签)

- **重新生成按钮点击:**
  1. 删除当前 AI 消息
  2. 重新发送上一条用户消息
  3. 显示加载指示器
  4. 接收新的 AI 回复

### 4.4 加载指示器 (LoadingIndicator)

```
┌─────────────────────────────────────────────────────┐
│  🤖 Assistant                                        │
│  ┌───────────────────────────────────────────────┐ │
│  │ ● ● ●  正在思考...                             │ │
│  └───────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────┘
```

**元素列表:**

1. **动画指示器** (AnimatedDots)
   - 内容: 三个点的动画
   - 动画: 依次闪烁或跳动

2. **状态文本** (StatusText)
   - 内容: "正在思考..." 或 "正在生成回复..."
   - 功能: 提示用户 AI 正在处理

**显示时机:**
- 用户发送消息后立即显示
- AI 开始返回内容后隐藏
- 如果支持流式响应,在流式输出时隐藏

### 4.5 空状态 (EmptyState)

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│                    [图标]                            │
│                                                     │
│              开始与 AI 对话                          │
│                                                     │
│         在下方输入框中输入你的问题                     │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**显示时机:**
- 应用启动且没有对话历史时
- 清空对话后

**元素列表:**
1. 引导图标
2. 引导文本
3. 提示文本

## 5. 输入区域 (InputArea)

### 5.1 元素构成

```
┌─────────────────────────────────────────────────────────┐
│  ┌───────────────────────────────────────────────────┐ │
│  │ 输入你的问题...                                    │ │
│  │ [多行文本输入框,可自动扩展高度]                     │ │
│  └───────────────────────────────────────────────────┘ │
│                                    [发送] [停止生成]    │
└─────────────────────────────────────────────────────────┘
```

**元素列表:**

1. **输入框** (InputBox)
   - 类型: 多行文本输入框 (textarea)
   - 占位符: "输入你的问题..."
   - 高度: 自适应
     - 最小高度: 80px (约 3 行)
     - 最大高度: 200px (约 10 行)
     - 超过最大高度后内部滚动
   - 功能: 接收用户输入

2. **发送按钮** (SendButton)
   - 位置: 右下角
   - 内容: "发送" 或发送图标
   - 状态:
     - 默认: 可点击
     - 输入框为空: 禁用
     - 正在生成: 隐藏

3. **停止生成按钮** (StopButton)
   - 位置: 右下角 (与发送按钮同位置)
   - 内容: "停止生成" 或停止图标
   - 状态:
     - 默认: 隐藏
     - 正在生成: 显示并可点击

**交互逻辑:**

### 5.2 输入框交互

**键盘事件:**

- **Enter 键:**
  - 单独按 Enter: 发送消息
  - Shift + Enter: 插入换行符
  - Ctrl/Cmd + Enter: 发送消息

- **输入时:**
  - 自动调整输入框高度
  - 内容为空时禁用发送按钮
  - 内容不为空时启用发送按钮

**粘贴事件:**
- 支持粘贴纯文本
- 自动去除格式

### 5.3 发送消息流程

```
用户点击发送按钮 或 按 Enter
    ↓
验证输入不为空
    ↓
禁用输入框和发送按钮
    ↓
隐藏发送按钮,显示停止按钮
    ↓
在对话区域添加用户消息
    ↓
清空输入框
    ↓
在对话区域显示加载指示器
    ↓
调用后端 send_message 命令
    ↓
接收 AI 回复
    ↓
隐藏加载指示器
    ↓
在对话区域显示 AI 消息
    ↓
启用输入框
    ↓
隐藏停止按钮,显示发送按钮
    ↓
自动滚动到最新消息
```

### 5.4 停止生成流程

```
用户点击停止按钮
    ↓
调用后端 stop_generation 命令
    ↓
隐藏加载指示器
    ↓
显示部分 AI 回复 (如果有)
    ↓
在消息末尾添加 "[已停止]" 标记
    ↓
启用输入框
    ↓
隐藏停止按钮,显示发送按钮
```

## 6. 对话框和提示

### 6.1 目录选择对话框 (DirectoryPicker)

**触发:** 点击工具栏的 "选择目录" 按钮

**类型:** 系统原生目录选择对话框 (Tauri 提供)

**流程:**
1. 打开对话框
2. 用户浏览并选择目录
3. 用户确认或取消
4. 如果确认,返回目录路径

**触发:** 点击工具栏的 "清空对话" 按钮

```
┌─────────────────────────────────────┐
│  确认清空对话                        │
├─────────────────────────────────────┤
│                                     │
│  确定要清空对话历史吗?               │

## 7. 状态管理

### 7.1 应用状态

```typescript
interface AppState {
  // 会话状态
  isSessionInitialized: boolean;
  workingDirectory: string | null;
  
  // 对话状态
  messages: Message[];
  isLoading: boolean;
  
  // UI 状态
  inputValue: string;
  error: string | null;
}
```

### 7.2 状态转换

**初始状态:**
```
isSessionInitialized: false
workingDirectory: null
messages: []
isLoading: false
inputValue: ""
error: null
```

**选择目录后:**
```
isSessionInitialized: true
workingDirectory: "/Users/xxx/project"
```

**发送消息时:**
```
isLoading: true
inputValue: "" (清空)
messages: [...messages, userMessage]
```

**接收回复后:**
```
isLoading: false
messages: [...messages, assistantMessage]
```

**发生错误时:**
```
isLoading: false
error: "错误消息"
```

## 10. 无障碍支持

### 10.1 键盘导航

**Tab 键:**
- 按顺序聚焦可交互元素
- 顺序: 选择目录 → 清空对话 → 输入框 → 发送按钮

**Enter 键:**
- 在按钮上: 触发点击
- 在输入框: 发送消息

**Esc 键:**
- 关闭对话框
- 取消当前操作

### 10.2 屏幕阅读器

**所有交互元素:**
- 提供 aria-label
- 提供 role 属性

**消息:**
- 标记为 role="log"
- 新消息自动朗读

**按钮:**
- 描述性标签
- 状态变化通知

## 11. 性能考虑

### 11.1 消息列表优化

**大量消息时:**
- 使用虚拟滚动 (可选)
- 只渲染可见区域的消息
- 减少 DOM 节点数量

**Markdown 渲染:**
- 使用高效的 Markdown 库
- 缓存渲染结果

### 11.2 输入框优化

**自动高度调整:**
- 使用 debounce 减少计算
- 避免频繁重排

**输入验证:**
- 实时验证但不阻塞输入
- 使用 debounce 减少验证次数

---

## 12. 边界情况处理

### 12.1 网络错误

**发送消息失败:**
1. 显示错误提示
2. 保留用户输入
3. 提供重试按钮

**连接超时:**
1. 显示超时提示
2. 停止加载动画
3. 允许重新发送

### 12.2 会话错误

**会话未初始化:**
- 禁用输入框
- 显示提示: "请先选择工作目录"

**会话过期:**
- 显示提示: "会话已过期,请重新选择目录"
- 清空对话历史

### 12.3 输入验证

**空消息:**
- 禁用发送按钮
- 不允许发送

**超长消息:**
- 显示字符计数
- 超过限制时显示警告

## 13. 未来扩展预留
